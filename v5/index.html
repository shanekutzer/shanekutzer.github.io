The Technology

Two technologies.
One platform.

DataVec is powered by JSMX and MNVKD — built from the ground up to do what existing platforms cannot.

JSMX — COMPILER

JavaScript → C.
Automatically.

JSMX is our ahead-of-time JavaScript compiler. It converts your code to optimized C before it ever runs. No JIT warmup. No interpreted overhead. Native performance from the first request.

  • Compiles JavaScript to native C binaries at deploy time
  • Supports WinterTC and Node.js interface extensions
  • Handles standard modules at build time — no runtime cost
  • End-to-end tested — output verified against real JS interpreter
  • Dynamic objects implemented as C structs for fast introspection
MNVKD — RUNTIME

Always warm.
Gets faster under load.

MNVKD is our persistent C runtime — an actor system that keeps your code resident in memory permanently. Unlike V8 isolates or Lambda containers, MNVKD doesn't sleep between requests. Under high load, it becomes more efficient.

  • Zero cold starts — runtime deployed whole, always running
  • Actor model — stateful processes without the complexity
  • Pipelined requests automatically aggregated under load
  • Per-process isolation — each deployment fully sandboxed
  • HTTP/2 and HTTP/3 natively — no proxy layer, no overhead
  • Atomic deployments via unix hardlinks — zero-downtime versioning
