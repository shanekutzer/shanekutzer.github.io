Quickstart

Create a project and deploy a small JavaScript function through the full DataVec path.

datavec login
datavec create api-demo
datavec add handler.js
datavec build
datavec deploy
datavec compare

Build with PRYZM

PRYZM analyzes JavaScript and prepares optimized C output before execution. The build report should show supported language features, warnings, and optimization notes.

Run on PÜLS

PÜLS executes the workload with warm residency and observability. The runtime view should show requests, cold starts, resident memory, CPU time, and latency.

Measure results

The compare command should help teams inspect current stack performance beside DataVec performance using the same workload.

Early access limits

Early docs should be clear about supported JavaScript features, supported APIs, package limitations, memory limits, runtime limits, and known issues. Technical users trust transparency.
