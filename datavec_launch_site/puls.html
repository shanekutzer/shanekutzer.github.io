Runtime philosophy

Stop moving computation unnecessarily.

Many cloud systems spend huge effort starting, stopping, relocating, and coordinating work. PÜLS is designed around the opposite idea: keep related work close and ready.

01

Zero cold start architecture

Workloads are designed to stay warm so requests do not wait for repeated initialization.

02

Persistent memory residency

State can stay in memory instead of being rebuilt from disk or remote services on every request.

03

Locality aware execution

Computation, data, state, and communication are kept close wherever the workload allows it.
